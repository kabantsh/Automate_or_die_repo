طيب لو انت جديد في مجال ال IT أو ليك باع في المجال و حابي تتعلم أكثر عن التكنلوجيا بتوجه عن ال IT Automation (أتمته تكنلوجيا المعلومات السلسلة دي ممكن تكون ليك)، الليلة ان شاء الله حنتكلم عن تشغيل الجهاز عن طريق الشبكة automate or die - Wake on Lan (WoL) 🌐🔌 ما هو تشغيل الجهاز عن طريق الشبكة؟ تشغيل الجهاز عن طريق الشبكة المحلية (Wake on Lan ) هو ببساطة ميزة موجودة في اللوحة الأم (motherboard) بتسمح ليك انك تشغل الجهاز عن طريق الشبكة المحلية (ethernet)، و طبعا هنا لازم الجهاز يكون مربوط بي سلك. خطوات تشغيل الجهاز عن طريق الشبكة : 1- بتفتحها في إعدادات البايوس (BIOS) 🖥️📟. 2- بتجيب عنوان الماك لي كرت الشبكة(MAC Address). 3- بتنزل برنامج التشغيل عن طريق الشبكة مثلا Etherwake أو أي برنامج آخر 📀🖥️. 4- بشتغل الجهاز من بعد 🤖🔌🖥️. أعمل متابعة للهشتاغ (#automate_or_die) عشان تشوف المنشورات الجديدة. هل شغلت جهاز عن طريق الشبكة قبل كده ؟ #wakeonlan #wol #itautomation #الحمدلله #محمد_أيمن

    Windows Core and GUI (Graphical User Interface) نسختان من نظام التشغيل Windows ولكنهم يختلفان في الطريقة التي يتم بها تقديم واجهة المستخدم والتحكم في النظام Windows Core:: هو إصدار من Windows Server يتميز بعدم وجود واجهة مستخدم رسومية. حيث يتم تثبيته كإصدار قاعدة للنظام الأساسي بدون واجهة المستخدم الرسومية التقليدية المعروفة بي GUI. يتم توفير Windows Core لتوفير نظام تشغيل أكثر صغرًا وأمانًا وأقل استهلاكًا للموارد، حيث يتم تعطيل معظم الخدمات والمكونات غير الضرورية للواجهة الرسومية. يتم التحكم في Windows Core عبر سطر الأوامر PowerShell أو بواسطة أدوات إدارة عن بُعد مثل Remote Server Administration Tools (RSAT) أو Windows Admin Center. ----------------------------------------- GUI (Graphical User Interface):: الإصدارات التقليدية من Windows تتميز بوجود واجهة مستخدم رسومية كاملة. توفر الواجهة الرسومية بيئة تفاعلية للمستخدمين للتفاعل مع النظام وتشغيل التطبيقات وإدارة الإعدادات. تتضمن الواجهة الرسومية في Windows بيئة سطح المكتب ومشغلات النوافذ وقوائم البدء والأيقونات ومراقبة المهام والإعدادات والأدوات الإدارية المرئية. يوفر الواجهة الرسومية سهولة الاستخدام والتفاعل للمستخدمين الذين يفضلون التعامل مع النظام من خلال واجهة رسومية اكثر سهولة . الاختلاف الرئيسي بين Windows Core و GUI هو وجود الواجهة الرسومية في الإصدارات التقليدية (GUI) وعدم وجودها في Windows Core. حيث يستخدم Windows Core عادة في بيئات السيرفرات حيث يكون الأداء والأمان واستهلاك الموارد مهمة أساسية، بينما تستخدم إصدارات Windows مع واجهة المستخدم الرسومية (GUI) في بيئات سطح المكتب والأجهزة الشخصية حيث يكون التفاعل المباشر والسهولة في الاستخدام أكثر أهمية. ----------------------------------------- Windows Core:: يستخدم عادة في بيئات السيرفرات حيث يكون الأداء والأمان واستهلاك الموارد أمورًا حاسمة. بعض الاستخدامات الشائعة لـ Windows Core. سيرفرات الشبكات: يعتبر Windows Core خيارًا شائعًا لتثبيت خوادم الشبكات حيث يتم تقديم الخدمات مثل سيرفرات الملفات وسيرفرات الطباعة وسيرفرات قواعد البيانات وسيرفرات التطبيقات. ويساعد Windows Core في تحقيق أداء أعلى واستهلاك منخفض للموارد على السيرفر. الحوسبة السحابية ومراكز البيانات: حيث يستخدم Windows Core في بنية الحوسبة السحابية ومراكز البيانات حيث يكون التركيز على تحقيق كفاءة الأداء وتحقيق مستويات عالية من الأمان والتوفير في استهلاك الطاقة والتبريد. سيرفرات الويب وسيرفرات التطبيقات: يمكن استخدام Windows Core لتشغيل سيرفرات الويب وسيرفرات التطبيقات التي توفر خدمات عبر الإنترنت. حيث يسمح Windows Core بتحقيق أداء محسّن واستهلاك للموارد المنخفض في هذه الأنظمة. سيرفر الأمان والشبكات: يستخدم Windows Core في بنية الأمان وسيرفرات الشبكات التي توفر وظائف الأمان الأساسية مثل جدران الحماية وسيرفرات الأمان وسيرفرات المراقبة. ويعزز Windows Core الأمان والاستقرار ويقلل من فتحات الأمان المحتملة. تجربة المطورين واختبار الأداء: يمكن استخدام Windows Core كبيئة تجريبية للمطورين واختبار أداء التطبيقات. حيث يسمح Windows Core بتشغيل الأجهزة الافتراضية والأجهزة الوهمية بشكل فعال ويساعد في تحقيق بيئة اختبار مستقرة وموثوقة. Windows Core:: يتطلب خبرة متقدمة في إدارة الأنظمة واستخدام سطر الأوامر PowerShell لإدارة وتكوين السيرفرات لهذا يحتاج إلى اشخاص خبراء في سطر الأوامر .
